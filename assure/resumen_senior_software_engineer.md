# Resumen de Temas - Senior Software Engineer

## Temas Evaluados en el Examen

### 1. Agile Software Development

#### Continuous Integration III
**Conocimiento requerido:**
- Experiencia práctica configurando servidor de integración continua
- Configuración con herramientas de gestión de configuración:
  - Ansible, Puppet, Chef
  - Configuración como código
- Configuración con contenedores:
  - Docker en CI/CD
  - Vagrant para entornos de prueba
  - Integración de contenedores en pipeline
- Setup de pipelines complejos
- Integración con múltiples herramientas
- Troubleshooting de problemas de CI

#### Continuous Delivery III
**Conocimiento requerido:**
- Experiencia práctica definiendo canalización de entrega continua (CD pipeline)
- Setup de Jobs/Stages:
  - Definición de etapas del pipeline
  - Condiciones y gates
  - Aprobaciones manuales cuando sea necesario
  - Rollback automático
- Herramientas específicas: Jenkins, GitLab CI, GitHub Actions, Azure DevOps
- Automatización completa del proceso de despliegue
- Estrategias de despliegue: Blue-Green, Canary, Rolling

### 2. Databases

#### Advanced Topics
**Conocimiento requerido:**
- **Big Data**: Conceptos, tecnologías (Hadoop, Spark), casos de uso
- **Database Replication**: 
  - Tipos: Master-Slave, Master-Master
  - Configuración y mantenimiento
  - Resolución de conflictos
  - Alta disponibilidad
- **Data Warehouse**: 
  - Conceptos de almacén de datos
  - ETL (Extract, Transform, Load)
  - Diseño dimensional (Star Schema, Snowflake Schema)
  - OLAP vs OLTP
- **Data Mining**: 
  - Conceptos de minería de datos
  - Técnicas y algoritmos
  - Aplicaciones prácticas

#### Tuning and Performance
**Conocimiento requerido:**
- Entender beneficios y cuidados en uso de índices:
  - Cuándo crear índices
  - Impacto en escritura vs lectura
  - Índices compuestos
  - Índices parciales
- Entender, analizar y optimizar execution plan:
  - Leer y entender execution plans
  - Identificar cuellos de botella
  - Table scans vs index seeks
  - Join algorithms (Nested Loop, Hash Join, Merge Join)
  - Query optimization techniques
- Optimización avanzada de consultas
- Análisis de rendimiento de base de datos

### 3. Object Oriented Programming

#### Principles III
**Conocimiento requerido:**
- Capacidad de generar componentes de software de alto nivel conformes a principios OOP
- Diseño de arquitecturas que cumplen SOLID
- Crear frameworks y librerías reutilizables
- Diseño de APIs públicas
- Aplicar principios en sistemas complejos
- Mentoría en principios de diseño

#### Design Patterns III
**Conocimiento requerido:**
- Aplicar y combinar patrones de diseño en proyectos existentes
- Entender efectos resultantes de aplicación de patrones:
  - Trade-offs de cada patrón
  - Cuándo NO usar un patrón
  - Combinación de múltiples patrones
- Refactorización usando patrones
- Crear nuevos patrones cuando sea necesario
- Documentar decisiones de diseño

#### Anti-patterns II
**Conocimiento requerido:**
- Identificar antipatrones en código existente
- Proveer soluciones mediante:
  - Uso de design patterns apropiados
  - Estrategias de refactorización avanzadas
  - Mejora de arquitectura
- Entender implicaciones profundas de antipatrones
- Prevenir antipatrones en diseño inicial
- Enseñar a otros sobre antipatrones

### 4. DevOps

#### Container Orchestration
**Conocimiento requerido:**
- Componentes de Kubernetes:
  - **Pods**: Unidad mínima de despliegue
  - **Services**: Exposición de servicios
  - **Deployments**: Gestión de aplicaciones
  - **ConfigMaps/Secrets**: Configuración
  - **Namespaces**: Aislamiento lógico
- Arquitectura de Cluster:
  - Master nodes (API Server, etcd, Controller Manager, Scheduler)
  - Worker nodes (kubelet, kube-proxy, container runtime)
  - Networking (CNI plugins)
- Administración de Cluster:
  - Escalado horizontal
  - Actualización de cluster
  - Gestión de recursos
  - Monitoring y logging
- Storage:
  - Persistent Volumes (PV)
  - Persistent Volume Claims (PVC)
  - Storage Classes
  - StatefulSets
- Containers y Workloads:
  - Deployments, ReplicaSets
  - StatefulSets para aplicaciones con estado
  - DaemonSets para nodos
  - Jobs y CronJobs
- Services, Load Balancing & Networking:
  - ClusterIP, NodePort, LoadBalancer, ExternalName
  - Ingress controllers
  - Network Policies
  - Service Discovery
- Security:
  - RBAC (Role-Based Access Control)
  - Pod Security Policies
  - Secrets management
  - Network policies
- Policies:
  - Resource quotas
  - Limit ranges
  - Pod disruption budgets
- Minikube: Kubernetes local para desarrollo

### 5. Software Architecture

#### Anti-patterns II (Arquitectura Avanzados)
**Conocimiento requerido:**
- Identificar cuando se incurre en antipatrones de arquitectura en sistemas:
  - **Autogenerated Stovepipe**: Código generado aislado
  - **Stovepipe Enterprise**: Sistemas empresariales aislados
  - **Cover Your Assets (CYA)**: Documentación excesiva sin valor
  - **Architecture By Implication**: Arquitectura implícita no documentada
  - **The Grand Old Duke of York**: Cambios innecesarios de arquitectura
- Análisis de sistemas existentes
- Propuesta de soluciones arquitectónicas
- Refactorización arquitectónica

## Puntos Clave para el Examen

1. **Experiencia práctica**: No solo teoría, sino experiencia real implementando soluciones
2. **Arquitectura**: Capacidad de diseñar sistemas complejos y escalables
3. **Optimización**: Entender y optimizar rendimiento en todos los niveles
4. **Liderazgo técnico**: Capacidad de guiar decisiones técnicas
5. **Resolución de problemas complejos**: Abordar problemas arquitectónicos y de diseño
6. **Mejores prácticas avanzadas**: Conocer y aplicar prácticas de nivel senior
7. **Trade-offs**: Entender compensaciones en decisiones técnicas
8. **Mentoría**: Capacidad de enseñar y guiar a otros desarrolladores

---

**Nota**: Este examen evalúa tu capacidad de diseñar, implementar y optimizar sistemas complejos de software, así como tu experiencia práctica en tecnologías avanzadas.
